Two Classic Program Goals:

  • Efficiency: Program takes minimal time and resources to run.

  • Correctness: Program runs in the way it is expected.

    • Software Testing and Verification:

    Kinds of tests:

    • Acceptance: Determines if the program properly completes the fuction it is designed to meet.
    • Intergration: Determines if the components of a system are capable of functioning as an intergrated whole.
    • Unit: Tests the smallest elements of code to determine if they function efficiently and correctly.
    • Functional: Detrmines if the code works according to its functional design.
    • Performance: Tests the stability, speed, and responsiveness of a program.
    • Regression: Tests the programs functionality after elements have been changed in response to prior tests.
    • Stress: Tests a program by pushing it to its extreme.
    • Usability: Tests the function of the software from the user interface.

    Software testing is done to detect:

    • Architectural flaws: Design choices in the fundamental elements of a code that make it difficult for other programmers to revise or test the code.
    • Poor design: Code that is not future proof; often overly complex.
    • Invalid functionality: Code fails to solve the problem it was designed to solve.
    • Security vunerabilities: Means for maliciously intent individuals to exploit the software.
    • Scalability: The ability of a piece to adapt to future demands; generally less code is more scalable code.

    Testing best practices:

    • Continous: Testing occurs during the entire development process.
    • Config management: Establishes and maintains consistency of the code througout its development/life cycle.
    • Service virtualization: Tests a a system replica to determine how well different modules intergrate.
    • Bug testing: Testing for undesired outcomes of code.
    • Metrics: The data of a program.
