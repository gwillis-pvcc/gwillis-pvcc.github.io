Software Life Cycle. Development Phase:
  • Requirements analysis
  • Design
  • Implementation
  • Testing
  • Process of determining the expectations of the user base.
  • Process of figuring out the softwares niche; what problems does the software seek to solve.
  • Process of installing the program and determining its bennchmarks according to designn objectives.
  • Process of debugging program.
Modular Implementation Software development is split into modules that can run a specific function independently.
Unified Modeling Language A standard notation for the creation of models of real-world objects.
Use Case Diagram A diagram of all possible interactions a user may hae with a software.
Data Relationships:
  • One to One
  • One to Many
  • Many to Many
  • A link between data in two data sets where each piece of data appears once in their respectve data sets.
  • A link between one parent data piece in a data set and many child data pieces in another data set.
  • A link between one or more data pieces in a dataset to one or more data pieces in another data set.
Design Pattern A reusable solution to common software dev problems.
Software Quality Assurance The monitoring of software dev to insure it meets international standards.
Software Testing:
  • Glass Box Testing
  • Black Box Testing
  • Beta Testing
  • Alpha Testing
  • Looks at the structure of the program and derives testable data from the programs code.
  • Examines the function of the program and its outputs without looking at the code.
  • Second phase in software testing where a sample group of the programs user-base tries it out.
  • First phase in software testing where the program function is tested by internal staff.
Documentation:
  • User Documentation
  • System Documentation
  • Technical Docummentation
  • Manuals/guides provided to the user to provide an easier user experience with the software
  • Documents that describe the system and its parts
  • Documents that describe the architecure of a software
Intellectual Property Law:
  • Software Liscence
  • Software Disclaimer
  • A legal liscence giving permission to use or distribute software.
  • A formal notice to the user of a software from the dev/liscence holder.
