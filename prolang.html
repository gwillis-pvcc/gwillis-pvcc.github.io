Chapter 6: Programming Languages
Term Definition
Machine language A programming language which a machine can directly read; binary for example.
Debugging The process of removing unwanted aspect from a program or piece of code.
Assembler A program that converts symbolic code into machine language.
Assembly language A mid-level language that is converted by an assembler.
Translator A program that converts one language to another language.
Compiler A program that converts code in a high-level language into machine language.
Interpreter A program that directly executes code as it is written without converting the code to machine language.
Cross-platform software Software designed to function in a multitude ofcomputing platforms; increasing its versatility and userbase.
Natural language A programming language that is written in spoken/written human-languages. Makes it easier for the programmer to code since the syntax is easy and natural to unserstand.
Formal language A laguage which is encoded via a formal system; does not rely on context. Formal languages rely on a syntax.
Machine independence Software that can be run on any machine.
High-level programming language A newer gen language which makes use of intuitive syntax; often based on human languages and allows the coder to code in their native language.
object-oriented programming A model of programming that relies on data/"objects", as opposed to logic/functions.
Scripting language A language where the code is interpreted as it is run, instead of before the code is run
